EVM Exporter 版本治理

工作台

围绕官方基线、企业分支、能力包和构建发布维护 exporter 企业版本。

版本治理关系

官方 upstream、company/ext、custom/custom.yaml 和企业发行版本

当前 Exporter

官方基线、本地版本、稳定扩展接口和装配清单

版本管理最佳实践 官方版本只作为 upstream 基线；企业版本只维护管理归属和当前本地版本；custom 能力包在发布运行时选择装配；cmg branch、company/ext、custom 目录、hook 和 lock 文件由系统生成，不手动填写。

企业版本列表

默认展示已保存企业版本，点击新增或编辑时再维护元数据。

Exporter 版本目录

统一维护官方 release 和手动上传的固定官方版本包。

新增企业版本

官方基线和企业管理信息分开维护，构建时再选择需要装配的能力包。

01

官方基线信息

选择已同步或手动上传的官方版本包，作为企业版本基线。

upstream/main -> cmg branch
02

企业版本信息

维护监控系统、部门、联系人、企业分支和稳定扩展接口。

系统自动维护： 转换规则：Exporter 选择 snmp_exporter 且企业信息不填写时，系统默认生成 cmg/snmp_exporter 分支和 snmp_exporter-internal-1.0.0 企业版本；custom/custom.yaml、company/ext hook 和构建锁定文件均由系统生成。

能力包资产库

能力包通过固定扩展点维护，可被不同企业版本在构建时选择装配。

扩展点规范

平台只暴露固定代码入口，能力包以源码方式在构建期编译进企业版 exporter。

新增能力包

只编辑平台暴露的代码区域，保存后进入能力包资产库，发布时再选择装配。
